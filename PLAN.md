## 1. v2 변경 배경

- 페이지 구조 3개 (온보딩/메인/설정). 온보딩 = 설정 페이지 공용.
- 공사명/작업원은 설정에서 "등록", 메인에서는 "선택"만.
- **"자동 입히기" 버튼 1개로 공통필드(GPS) + 사진 선택 플로우 전체 통합.**
- 사진은 **앨범에서 선택** (카메라 촬영 X, `capture` 속성 금지).
- "기존과 동일" 자동 감지 (공사명+작업원 매치 시 사업소/작업장소 프리필).
- Web Share 제거 → 로컬 저장 + 밴드 앱 열기.

## 2. 페이지 구조

- **설정 페이지 (= 온보딩)** — 첫 방문자(`workers.length === 0`) 자동 진입. 메인에서 "설정" 버튼으로도 진입.
- **메인 페이지** — 공통필드 + 자동 입히기 + 수동 사진 + 합성 + 공유.
- 단일 `index.html` 내 섹션 전환, hash 라우터. DOM id는 Phase 6(Claude Design) 대비 유지.

## 3. 데이터 모델 (localStorage `ami-board-state` v2)

```ts
interface StateV2 {
  version: 2;
  projectNames: string[];            // 등록된 공사명
  workers: string[];                  // 등록된 작업원 (순서 유지, 1→2→3→4번)
  lastSelected: {
    projectName?: string;
    workers?: string[];
    office?: string;                  // 테스트: "마포용산지사" 고정
    workplace?: string;
    workplaceCoord?: { lat: number; lng: number };
  };
  sessionHistory: Array<{             // 최근 30개, FIFO
    projectName: string;
    workers: string[];                // sort 후 저장
    office: string;
    workplace: string;
    workplaceCoord?: { lat: number; lng: number };
    timestamp: string;                // ISO 8601
  }>;
}
```

**v1 → v2 마이그레이션** (Phase A 구현 완료)
- `commonFields.projectName` → `projectNames = [값]` + `lastSelected.projectName`
- `workerHistory` → `workers` (중복 제거, 순서 유지)
- `savedCrews`, `recentSessions` → 삭제
- `version: 2` 세팅

## 4. 설정/온보딩 페이지 (Phase A 구현 완료)

- 모드 구분: `workers.length === 0` 여부
- 공사명 목록/추가/삭제
- 작업원 목록/추가/삭제/순서 변경 (▲▼)
- 온보딩 모드: 공사명 ≥1 + 작업원 ≥1 → "완료" → 메인
- 설정 모드: "메인으로" 버튼 상시
- Enter 키로 추가 (IME 조합 중 제외)
- **"완료" 클릭 시 입력 필드 자동 flush**: 공사명/작업원 입력칸에 텍스트가 남아 있으면 완료 핸들러가 내부적으로 `addProject()` / `addWorker()` 호출 후 활성화 조건 재평가. "추가" 한 번 더 누르는 UX 부담 제거 (Phase E).

### 5.2 작업장소 입력 방식 (보조)

- **GPS 버튼**: 현재 좌표 → 카카오 역지오코딩 (v1 재사용)
- **지도 모달**: "지도에서 선택" 버튼 → 카카오 지도 오픈 → 단일 탭으로 핀 드롭 → 좌표 역지오코딩 → "확인"
- 수동 텍스트 수정 가능
- 이 3경로는 자동 입히기와 병행 (자동 입히기 실패 시 폴백, 수정 시 보조)

### 5.3 "기존과 동일" 자동 감지

- 공사명 또는 작업원 선택 변경 시 훅 재실행
- `sessionHistory`에서 `projectName` 일치 + `workers`(정렬 후) 일치 최신 엔트리
- 매치: 사업소/작업장소/좌표 프리필 + "이전 세션에서 자동입력됨 / 초기화" 배너
- 매치 없음: 공백 유지. 자동 입히기 시 GPS로 채움

### 5.4 "자동 입히기" 통합 플로우 (Phase E 개편)

단일 버튼 **"자동 입히기"** 클릭 시 2단계:

**1단계: 공통필드 자동 채우기** (기존 Phase C 유지)
- "기존과 동일" 매치 먼저 확인 (공사명+작업원 조합)
  - 매치: 해당 세션의 사업소/작업장소/좌표 프리필
  - 매치 없음: GPS 획득 → 역지오코딩 → 작업장소 입력 + 사업소 매핑
    - 테스트 단계: 사업소 마포용산 고정
    - 실운영: 주소 "구" 기반 7개 지사 매핑 테이블 (별도 준비)
- GPS 실패 시: "작업장소를 수동 입력하세요" 알림 후 2단계로 진행
- **실행 순서 (iOS/Safari user gesture 보존)**: GPS는 버튼 클릭 직후 fire-and-forget 시작 → 사진 피커를 동기 경로 최하단에서 즉시 열기 → 사용자가 사진 고르는 동안 GPS 백그라운드 진행 → 선택 완료 후 GPS 결과 반영

**2단계: 사진 슬롯 UI (한 화면에 동시 배치 — Phase E)**

v1/Phase C의 **순차 모달** (작업원 → 서류 → 차대비1 → 차대비2) 은 폐기. 한 화면에 **N+2 개 슬롯**을 카드 그리드로 보여줌:

- 슬롯 1: **작업원** — 태그 = 전체 선택 작업원 공백 연결
- 슬롯 2: **서류** — 태그 = 동일
- 슬롯 3~N+2: **{이름} 차대비** — 각 작업원별 1장, 태그 = 해당 이름만

각 슬롯 구성:
- 라벨 (예: "우영준 차대비")
- [사진 선택] 버튼 → `<input type="file" accept="image/*">` 프로그래매틱 트리거 (`capture` 속성 없음, 앨범에서 선택)
- 선택 후 썸네일 즉시 렌더 + [재선택] / [건너뛰기] 버튼
- 건너뛰기 시 해당 슬롯 합성 안 함

**하단 "합성 미리보기" 버튼**: 모든 슬롯이 "선택" 또는 "건너뛰기" 상태이면 활성화.
- 건너뛴 슬롯 제외하고 compose() 일괄 적용 → 프리뷰 그리드로 교체

**(iOS 주의)** 파일 input은 OS 정책상 "사진 찍기 / 사진 보관함 / 파일 선택" 액션시트를 띄움. 웹 JS로는 완전 스킵 불가. 본 UX에서는 슬롯마다 한 번씩 "사진 보관함" 탭이 필요. PWA 홈 화면 추가 시 일부 디바이스에서 포토 피커 바로 뜨는 경우 있음 (별도 검증).

**※ 재클릭 시**: "자동 입히기" 버튼을 플로우 진행 중 다시 누르면 `confirm("진행 중인 자동 입히기를 초기화하시겠습니까?")` 네이티브 확인 → 확인 시 상태 리셋 후 1단계부터 재시작.

**3단계: 합성 미리보기 + 저장 + 밴드** (기존 Phase C/D 그대로)
- 프리뷰 그리드 렌더, 개별 "다시 선택" 지원
- "저장 + 밴드 열기" 버튼 → 다운로드 + 딥링크 (Phase D)

## 7.1 Phase F 상세 (실제 워크플로우 반영)

**실제 워크플로우** (2026-04-19 영준님 확정)
- 아침에 조원 모여 **총 3~5장 사진을 한 번에 촬영** (작업원 단체 + 서류 + 개인별 차대비)
- 관리자가 밴드에 공지글 "작업전 안전점검 내용(사진.체크리스트)"를 선행 게시 (날짜별 1건)
- 각 조는 공지글 **댓글로** 사진 첨부 (오전 1회 + 오후 1회)
- 밴드 댓글 자동 첨부는 iOS/Android 웹 불가 → 앱은 **사진첩 저장까지만** 책임

**UI 플로우**
1. 공통필드 채움 + "자동 입히기" 클릭
2. **1단계 GPS/역지오코딩** (기존)
3. **2단계 멀티셀렉트 1회**: `<input type="file" accept="image/*" multiple>` → 3~5장 한 번에 선택 (앨범)
4. **3단계 슬롯 할당 UI**: 선택한 사진 썸네일 그리드 + 슬롯 목록(작업원/서류/{이름} 차대비). 각 썸네일을 슬롯에 탭-할당 (또는 드래그). 1장을 여러 슬롯에 배정 가능. 남은 슬롯은 "건너뛰기"
5. **4단계 합성**: 할당된 사진만 compose() 일괄 처리, 프리뷰 그리드
6. **5단계 사진첩 저장**: "사진첩 저장 + 밴드 열기" 버튼
   - `navigator.share({ files })` → iOS 공유시트 → "이미지 저장" 1탭으로 bulk 저장
   - Web Share 미지원 시 폴백: `<a download>` 다회 (현 Phase D 로직)
7. **6단계 밴드 앱 열기**: 딥링크 `bandapp://` (iOS) / `intent://...` (Android). 공지글 URL 스킴 존재 시 해당 글로 직행 시도, 실패 시 밴드 홈
8. 사용자 수동: 밴드에서 공지글 찾아 → 댓글 영역 → 사진첩에서 저장된 N장 선택 → 등록

**스코프 외 (자동화 불가, 명시)**
- 밴드 공지글 댓글 자동 첨부
- 밴드 공지글 직접 열기 (URL 스킴 미검증)
